================================================================
  PLANIFICATEUR DE MARIAGE — Mettre l'application en ligne
  (pour fournir un LIEN navigateur aux clientes, en plus du fichier)
  Par La Fabrique Astucieuse
================================================================

POURQUOI ?
----------
Sur iPhone, ouvrir le fichier HTML depuis l'apercu d'un e-mail affiche
souvent une page vide (l'apercu Mail / Quick Look bloque le JavaScript).
La solution la plus fiable pour le mobile est de fournir un LIEN
(une adresse https://...) que la cliente ouvre dans Safari ou Chrome.

Le fichier "index.html" fonctionne :
  - en local sur ORDINATEUR (double-clic) ............ OK
  - heberge en ligne via un LIEN (mobile + ordi) ..... OK (recommande mobile)

Le fichier est 100 % autonome : aucune dependance externe, aucun
serveur, aucune base de donnees. Il s'heberge donc tel quel.


================================================================
OPTION A — GitHub Pages (gratuit, durable)
================================================================
1. Creer un compte sur https://github.com (gratuit).
2. Cliquer sur "New repository".
     - Nom : par exemple  wedding-planner
     - Cocher "Public"
     - Valider (Create repository).
3. Cliquer "uploading an existing file" (ou Add file > Upload files).
4. Glisser le fichier  index.html  (celui de ce dossier).
5. Cliquer "Commit changes".
6. Aller dans  Settings  (onglet du depot) > Pages (menu de gauche).
7. Section "Build and deployment" :
     - Source : "Deploy from a branch"
     - Branch : main   /   (root)   puis Save.
8. Patienter 1 a 2 minutes. L'adresse apparait en haut de la page Pages :
     https://VOTRE-NOM.github.io/wedding-planner/
   C'est le LIEN a donner aux clientes.

   (Le fichier s'appelant index.html, l'adresse se termine par "/"
    et ouvre automatiquement l'application.)


================================================================
OPTION B — Netlify Drop (le plus rapide, sans compte au depart)
================================================================
1. Aller sur  https://app.netlify.com/drop
2. Glisser-deposer le fichier  index.html  (ou tout le dossier).
3. Netlify genere immediatement un lien du type :
     https://nom-aleatoire-123.netlify.app/
4. (Optionnel) Creer un compte gratuit pour conserver le lien et
   le renommer dans Site settings > Change site name.


================================================================
OPTION C — Autre hebergement statique
================================================================
N'importe quel hebergement de "site statique" convient :
Cloudflare Pages, Vercel, Firebase Hosting, OVH (dossier www), etc.
Il suffit d'y deposer  index.html  a la racine.
L'adresse type sera :  https://mon-site.com/wedding-planner/index.html


================================================================
CE QU'IL FAUT SAVOIR (a transmettre eventuellement aux clientes)
================================================================
- Les donnees restent enregistrees DANS LE NAVIGATEUR de l'appareil
  et du navigateur utilises (localStorage). Elles ne sont PAS envoyees
  en ligne, meme avec un lien : l'hebergement ne sert qu'a OUVRIR
  l'application, pas a stocker les donnees.

- Chaque appareil/navigateur a ses propres donnees. Pour transferer,
  utiliser Exporter / Importer un JSON (onglet Sauvegarde).

- Les pieces jointes (page Documents) restent dans le navigateur
  (IndexedDB) et ne sont pas incluses dans le JSON. Garder les
  originaux ailleurs.

- Bien preciser dans la fiche Etsy / le message de livraison :
  "Sur telephone, ouvrez le LIEN dans Safari ou Chrome.
   N'ouvrez pas le fichier depuis l'apercu d'un e-mail."
